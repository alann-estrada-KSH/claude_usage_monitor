Privacy Policy — Claude Usage Monitor

Last updated: July 6, 2026

Claude Usage Monitor is an unofficial, independent tool. It is not affiliated with, endorsed by, or built by Anthropic. It shows the usage percentages already visible on your own claude.ai/settings/usage page in a small dashboard/tray app.

Short version

This app collects nothing and sends nothing anywhere except claude.ai and, optionally, GitHub (for update checks). There is no backend server owned by this app, no analytics, and no advertising.

What the app does with your Claude.ai session

  • You log in to claude.ai through a real, native WebView — the same login flow you'd use in a browser.
  • The resulting session cookie is stored by the operating system's own WebView engine (Android WebView, or the OS-native WebView on Linux/Windows). This app never reads, copies, exports, or transmits that cookie to any server it controls.
  • On a timer, the app uses that existing session to call claude.ai's own usage-data endpoint and read your 5-hour/weekly usage percentages. This traffic goes directly from your device to claude.ai, the same as if you opened that page in a browser yourself.

What's stored locally

  • Account labels you choose (e.g. "Work", "Personal") and the last-seen usage percentages/reset times, so the dashboard has something to show immediately on open.
  • Your app preferences (theme, refresh interval, notification thresholds, etc).
  • A small log of which notifications have already fired, so you don't get duplicate alerts.

All of this is stored only on your own device (via Hive, a local key-value store) and is never uploaded anywhere. Uninstalling the app deletes it.

Notifications

Local notifications (limit resets, limit exhaustion) are generated and shown entirely on-device. No notification content is sent to or generated by any server.

Android background refresh

On Android, an optional, user-toggled setting periodically pings claude.ai in the background (via Android's WorkManager) purely to keep your session from expiring due to inactivity. This is the same kind of request as a normal usage check — it goes to claude.ai only, not to any server run by this app.

Update checks (Windows)

On Windows, the in-app "Check for updates" feature queries the public GitHub Releases API for this project to see if a newer version exists. This is an unauthenticated request to GitHub containing no personal data. Linux updates go through a standard apt repository instead, with no in-app network request at all.

Claude status checks

The app also periodically checks Anthropic's public status page (status.claude.com) so it can tell you "Claude itself is down" instead of misreporting your own account as broken. This is a public, unauthenticated request with no personal data attached.

No analytics, no ads, no third-party trackers

This app contains no analytics SDKs, no crash reporting services, no advertising, and no third-party trackers of any kind.

Data sharing

Nothing is shared with third parties, because nothing about your usage of this app is collected in the first place.

Changes to this policy

If this policy changes, the update will be posted at this same URL with a new "Last updated" date.
